Flow

RemUI payment flow has three steps:

StepWhat happens
1UI sends onClick: paym("gateway", "/ui/payment/confirm", {...}).
2Dart click handler routes this to the registered gateway in RemUI.init(...paymentGateways).
3Gateway result calls await RemUI.paymentCallback({...}); RemUI posts to callback path from paym(...).

Dart Init: Register Gateways

RemUI.init(
  "http://localhost:3000",
  {"mediaQuery": true},
  paymentGateways: {
    "razorpay": (j) async {
      // Launch SDK checkout here, then send final result.
      await RemUI.paymentCallback({
        "status": "success",
        "gateway": "razorpay",
        "paymentId": "pay_123",
        "orderId": "order_123",
        "amount": j["data"]?["amount"],
        "note": j["data"]?["note"],
      });
    },
    "stripe": (j) async {
      // Open Stripe sheet or hosted page.
      await RemUI.paymentCallback({
        "status": "success",
        "gateway": "stripe",
        "intentId": "pi_123",
        "amount": j["data"]?["amount"],
      });
    },
    "paypal": (j) async {
      // Open PayPal approval flow.
      await RemUI.paymentCallback({
        "status": "success",
        "gateway": "paypal",
        "captureId": "CAPTURE_123",
        "amount": j["data"]?["amount"],
      });
    }
  }
);
Use the same key names in paymentGateways as in paym("method", ...).

Server Side: onClick with paym(...)

import { ui, paym } from "../../../import";

const checkoutButton = ui("ElevatedButton", {
  child: ui("Text", { text: "Pay Now" }),
  onPressed: paym("razorpay", "/ui/payment/confirm", {
    amount: 19900,
    currency: "INR",
    note: "Booking #A1003",
    bookingId: 1003,
    customerId: 77
  })
});

Gateway Example: Razorpay

Typical payload fields sent from UI to gateway handler, then back to callback route:

// paym payload (request side)
{
  "type": "payment",
  "gateway": "razorpay",
  "callback": "/ui/payment/confirm",
  "data": {
    "amount": 19900,
    "currency": "INR",
    "note": "Booking #A1003",
    "bookingId": 1003
  }
}

// paymentCallback payload (result side)
{
  "status": "success",
  "gateway": "razorpay",
  "paymentId": "pay_xxx",
  "orderId": "order_xxx",
  "signature": "sig_xxx"
}

Generic Callback Route (Express + Prisma)

import type { PrismaClient } from "@prisma/client";
import type { Request, Response } from "express";
import { sendBack, navReplace, SnackBar } from "../../../import";

export default {
  path: "/ui/payment/confirm",
  method: "POST",
  auth: "none",
  run: async (req: Request, res: Response, prisma: PrismaClient) => {
    const body = req.body ?? {};
    const gateway = String(body.gateway ?? "");
    const payment = body.data ?? {};
    const requestData = body.request ?? {};

    const status = String(payment.status ?? "failed").toLowerCase();
    const bookingId = Number(requestData.bookingId ?? 0);

    if (status !== "success") {
      sendBack(res, [
        SnackBar("Payment failed. Please try again."),
      ]);
      return;
    }

    // Save payment in DB (simplified)
    await prisma.payments.create({
      data: {
        bookingId,
        gateway,
        amount: Number(requestData.amount ?? 0),
        reference: String(payment.paymentId ?? payment.intentId ?? payment.captureId ?? ""),
        rawJson: JSON.stringify(body),
      },
    });

    sendBack(res, [
      SnackBar("Payment successful"),
      navReplace(`/ui/booking/success/${bookingId}`),
    ]);
  },
};

Tips

  • Always validate payment signatures/tokens server-side before marking success.
  • Treat amount and currency from the server as authoritative.
  • Use idempotency keys for callback routes to prevent duplicate payment records.
  • If payment is canceled, still call RemUI.paymentCallback({...}) with failure status for consistent flow.
