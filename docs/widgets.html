Widgets Reference

Registry-driven widget guide for RemUI. Every widget below is sourced from registers.dart.

Notes

Brace interpolation works on any string field now, including text, URLs, labels, and other JSON string values. Missing values stay unresolved so Foreach can hide undeclared sources cleanly.
Foreach uses $name$-style row placeholders. Use {vars} for normal server/runtime values and $row.fields$ inside repeated templates.
Navigation supports both page pushes and modal dialogs. Use nav('/ui/path') for pages and nav('/ui/path', 'dialog') for overlays that keep the background page visible.
remSharedPref removes one or more SharedPreferences keys from callback payloads.
JSON-first Backend-built Flutter-rendered

Metadata

Every widget node now has chainable metadata helpers. Use .meta() for page metadata such as title and description, and .tag() / .alt() for SSR-friendly HTML hints on regular widgets.

ui('Scaffold', {
  body: ui('Text', { text: 'Hello' })
})
  .meta('title', 'Hello Page')
  .meta('description', 'SSR demo');

Initialization

Use RemUI.init to set the backend URL, runtime config, custom widget registers, custom pages, and page action handlers.

RemUI.init(
  "http://localhost:3000",
  {"mediaQuery": true},
  registers: {
    "TextO": (j) => ui('Text', { text: j["text"]?.toString() ?? '' })
  },
  pages: {
    ".page_cd_a": (j) => PageCDA(j)
  },
  pageFunction: {
    ".main": (onClick, url) {
      if (url == "/next") {
        RemUI.changePage("/ui/next");
      }
    }
  }
);
Custom registers override built-in widget types when the names match. Custom pages can be Flutter-only widgets and do not need to come from the RemUI server.

Produced by equate(). The renderer treats the root and chained elseIf nodes as a normal if/else-if chain.

FieldTypeMeaning

Foreach

Repeats a template for every item in a remote response or inline JSON variable. Use $field$ inside the repeated template so it does not conflict with server-side {vars} interpolation.

FieldTypeMeaning
urlstringRemote request or inline JSON source. Example: GET:https://api... or {abd}.
toForeachstringPath to the array. Use data for API responses or . for root arrays.
datawidgetTemplate repeated for each item.
loadingwidgetShown while the request is in flight.
fallbackwidgetShown when the source is missing, empty, or invalid.
spacingnumberGap between repeated items.
ui('foreach', {
  url: 'GET:http://localhost:3000/api/demo/users',
  toForeach: 'data',
  data: ui('Text', { text: '$name$ - $role$' })
});

ui('foreach', {
  url: '{abd}',
  toForeach: '.',
  data: ui('Text', { text: 'Heres $char$' })
});

Timeout

Deferred callback executor widget. Useful for delayed state updates, preference writes, or post-render callback orchestration.

FieldTypeMeaning
dataobject | listCallback payload (for example sendBack() output).
timeout / duration / msnumberOptional delay before running data.
childwidgetOptional child rendered while waiting.

  data: remui
    .sendBack()
    .timeout(1000)
    .setVar('status', 'ready')
    .setSharedPref('status=ready')
});
