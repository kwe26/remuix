WebView Callback Guide

Open a webview from a callback, watch URL changes, and redirect back into RemUI when a match occurs.

Sections Overview Builder API Match Modes RemUI Redirect Server Usage Flutter Setup Execution Flow Examples FAQ

Overview

WebViewCb() creates a callback payload that opens an in-app webview. It is designed for flows where a third-party page eventually redirects to a known URL, and that URL should hand control back to RemUI.

In-app webview URL match RemUI redirect Back-to-main behavior
This is a callback feature, not a standard widget. You attach it with callback.addWebView(webView) or sendBack().addWebView(webView).

Builder API

const webView = WebViewCb()
  .url('https://checkout.example.com/start')
  .fullscreen()
  .match('example.com', 'contains')
  .remuiContains('/ui/payment/confirm');
MethodMeaning
.url(value)Initial URL loaded into the webview.
.fullscreen(true)Opens the webview full screen. If omitted, a centered dialog is used.
.match(value, mode)Adds a URL matcher. Multiple matches are allowed.
.remuiContains(path)RemUI route to open once a match is detected.
.addWebView(webView)Adds the webview callback to a callback builder.

The full callback payload is serialized as JSON and executed on the Flutter side.

Match Modes

.match() supports three modes.

ModeBehaviorExample
containsMatches when the current URL contains the provided string.match('example.com', 'contains')
exactMatches only when the URL is exactly equal to the provided string.match('https://example.com/done', 'exact')
startsWithMatches when the URL starts with the provided string.match('https://example.com/return', 'startsWith')
You can add multiple .match() calls. The first one that matches wins.

RemUI Redirect

When a match fires, the webview closes and RemUI opens the route supplied by .remuiContains() using replace-style navigation.

WebViewCb()
  .url('https://payments.example.com/start')
  .match('payments.example.com/success', 'contains')
  .remuiContains('/ui/payment/confirm');

The Flutter runtime opens /ui/payment/confirm?from=<decoded-value>. If the matched URL already contains a from query parameter, that value is decoded and forwarded. Otherwise the matched URL itself is decoded and used as the from value.

The opened RemUI page behaves like a replacement page. Pressing back returns to /ui/main.

Server Usage

Attach the webview payload to any callback builder. The recommended pattern is to send it from a callback response, then let the client open the webview immediately.

app.post('/ui/start-payment', (req, res) => {
  const callback = ui.getCallback(req, res, '#cb1001');

  const webView = WebViewCb()
    .url('https://checkout.example.com/start?order=ABC123')
    .fullscreen()
    .match('checkout.example.com/complete', 'contains')
    .remuiContains('/ui/payment/confirm');

  callback
    .addWebView(webView)
    .send();
});

You can also combine the webview payload with other callbacks. For example, close dialogs, set variables, or persist preferences after the webview returns.

callback
  .addWebView(webView)
  .setVar('paymentStatus', 'pending')
  .setSharedPref('lastPaymentGateway=webview')
  .send();

Flutter Setup

The Flutter side already understands the callback payload and opens the webview using webview_flutter.

RemUI.init(
  'http://localhost:3000',
  {'mediaQuery': true},
  paymentGateways: {
    'razorpay': (j) async {
      await RemUI.paymentCallback({
        'status': 'success',
        'gateway': 'razorpay',
        'paymentId': 'pay_123',
      });
    },
  },
);

For the webview flow specifically, the runtime watches the URL. When it matches, it closes the webview and opens the RemUI target page.

Execution Flow

StepWhat happens
1Backend sends a callback with webView payload.
2Flutter opens the webview, either fullscreen or dialog mode.
3User completes the external flow and the URL changes.
4The runtime checks each .match() rule against the current URL.
5On match, the webview closes and RemUI opens the configured route with ?from=....
6Back from that RemUI page returns to /ui/main.

Examples

Checkout flow

const webView = WebViewCb()
  .url('https://checkout.stripe.com/pay/cs_test_123')
  .fullscreen()
  .match('checkout.stripe.com', 'contains')
  .remuiContains('/ui/payment/confirm');

callback.addWebView(webView);

Approval flow with exact match

const webView = WebViewCb()
  .url('https://accounts.example.com/approve')
  .match('https://accounts.example.com/success', 'exact')
  .remuiContains('/ui/accounts/success');

Multiple match rules

const webView = WebViewCb()
  .url('https://example.com/login')
  .match('example.com/return', 'contains')
  .match('example.com/complete', 'startsWith')
  .remuiContains('/ui/complete');

FAQ

Does this open a new browser tab?

No. It opens inside the Flutter app using the in-app webview.

What does from contain?

The matched URL, decoded from base64 when applicable. If the URL already includes a from parameter, that value is preferred.

Why does back return to /ui/main?

The RemUI page opened from the webview is marked as a replace-style route with a from marker. The page intercepts back navigation and sends the user home.

Can I use it outside payments?

Yes. Any external flow that ends in a predictable return URL can use the same callback payload.
